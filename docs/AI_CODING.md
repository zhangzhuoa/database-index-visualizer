# AI Coding 辅助开发说明

本文档说明本项目如何使用 AI Coding 完成需求拆解、开发实现、测试验证、文档整理和上线准备。项目本身不调用大模型 API，也不包含 AI 推理功能；它的定位是“使用 AI Coding 辅助完成的数据库索引结构可视化工程项目”。

## 使用 AI 的方式

AI 在本项目中主要承担工程协作角色：

- 将“大而模糊”的目标拆解为可执行任务，例如从 B+ 树 MVP 扩展到多索引架构，再整理为 v1.0 在线展示版。
- 约束架构边界，例如核心算法不能写进 React 组件，新增索引必须通过统一 `IndexEngine` 接入。
- 辅助设计通用类型、动画事件、ViewModel 和 SVG 渲染组件之间的数据流。
- 辅助补充测试，覆盖核心算法、engine 适配、ViewModel 输出和关键动画事件。
- 辅助整理 README、架构文档、CHANGELOG、ROADMAP 和部署说明。
- 辅助进行上线前检查，例如本地构建、测试、静态部署配置和简历展示材料整理。

## 人工参与和验收

项目开发中保留人工判断和验收：

- 由开发者确定项目定位：纯前端、无后端、无数据库、无用户系统。
- 由开发者确认五种索引结构的学习展示范围，而不是继续无限扩展新结构。
- 由开发者在浏览器中完整验收五种索引的切换、示例、播放控制和 reset 行为。
- 由开发者完成 GitHub 上传、Vercel 部署和线上访问检查。
- 由开发者决定如何把项目包装为面试项目，而不是把它描述成“AI 自动生成项目”。

## 架构约束

开发过程中始终遵守以下约束：

- 保持 React + TypeScript + Vite 技术栈。
- 保持纯前端静态网站，不新增后端服务。
- 不连接数据库，不引入用户登录。
- 图形渲染以 SVG 为主，不引入 D3、React Flow、Cytoscape 等复杂图库。
- 核心索引算法与 React 组件分离。
- 可视化渲染逻辑与核心索引逻辑分离。
- 新增索引结构必须接入统一 `IndexEngine`。
- 修改公共架构时必须保护已有 B+ 树行为。

## AI Coding 工作流

### 1. 需求拆解

将项目拆成多个阶段：

- B+ 树 MVP
- 多索引架构抽象
- 哈希索引接入
- R 树接入
- 四叉树接入
- 网格索引接入
- v1.0 在线展示版整理

每个阶段都明确“不新增后端、不连接数据库、不引入复杂图库”的边界。

### 2. 架构设计

核心设计围绕四层展开：

```text
IndexEngine
  -> 核心索引算法
  -> AnimationEvent
  -> ViewModel
  -> SVG 渲染组件
```

页面层只做：

- 索引类型切换
- 用户输入解析
- 操作分发
- 动画播放
- 示例载入
- 高亮状态管理
- SVG 组件组合

### 3. 测试补充

使用 Vitest 覆盖：

- B+ 树插入、查找、范围查询和叶子链表顺序
- 哈希索引 bucket 映射和冲突链
- R 树 MBR 扩张、节点分裂和空间范围查询
- 四叉树空间划分、边界处理和范围查询
- 网格索引 cell 映射、边界处理和范围查询
- `IndexEngine` 多实现的类型和 reset 行为
- v1.0 内置演示案例和操作支持矩阵

### 4. 文档整理

AI 辅助把开发过程中的任务记录整理为面向读者的文档：

- `README.md`：面试官和用户的第一入口。
- `docs/ARCHITECTURE.md`：说明工程分层与数据流。
- `docs/AI_CODING.md`：说明 AI Coding 在项目中的实际作用。
- `CHANGELOG.md`：记录版本演进。
- `ROADMAP.md`：说明 v1.0 完成状态和后续可选扩展。

### 5. 上线准备

上线前执行：

```bash
npm test
npm run build
```

部署方式选择 Vercel，构建命令为 `npm run build`，输出目录为 `dist`。

## 面试中可以强调的点

- 这不是简单页面，而是一个有统一接口、测试和部署的完整前端工程。
- AI 的价值体现在“协助工程化推进”，不是替代开发者判断。
- 项目通过明确架构边界避免了算法逻辑和 React 组件耦合。
- SVG 可视化由 ViewModel 驱动，动画说明由 `AnimationEvent` 驱动。
- v1.0 收口时没有继续堆功能，而是补齐示例、交互、测试、文档和部署。

## 面试中不建议夸大的点

- 不要说项目内置了 AI 功能。
- 不要说项目连接真实数据库。
- 不要说使用了真实数据库执行计划。
- 不要把它包装成后端全栈系统。

更准确的说法是：

```text
这是一个使用 AI Coding 辅助完成的数据库索引结构可视化工具，重点展示前端工程化、数据结构可视化、统一架构抽象和测试验证能力。
```
